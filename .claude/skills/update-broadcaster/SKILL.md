# Update Broadcaster

## When to Use
- After completing meaningful work (new features, research findings, plan changes).
- Before sending status updates, newsletters, or posting build-in-public content.

## Workflow
1. Review `docs/progress.md`, recent commits, and relevant knowledge entries.
2. Capture highlights: what changed, why it matters, next steps, blockers.
3. Tailor the output to audience (investor, teammate, social media). Include calls-to-action if needed.
4. Store the update draft in `knowledge/projects/<project>-updates.md` or share via preferred channel.
5. Link the update from Level 0/2 docs when appropriate.

## Tips
- Keep language concise and outcome-focused.
- Mention metrics or tests passed where relevant.
- Note follow-up actions so future updates can reference outcomes.
